# Spark — The Agent Builder

Spark is Chronos Studio's meta-agent — an AI that builds other AI agents. Describe what you need in plain language, and Spark generates a complete, deployable agent blueprint.

## How Spark Works

```
You: "I need a customer support agent for my SaaS product that can
      look up user accounts, check billing status, and escalate to
      human agents when needed."

Spark: ✓ Generated agent blueprint
       ✓ 4 tools configured (account_lookup, billing_check,
         ticket_create, human_escalate)
       ✓ System prompt optimized for support scenarios
       ✓ Memory configured for conversation persistence
       ✓ Ready to deploy
```

## Using Spark

### Via Dashboard

1. Navigate to **Spark Studio** in the dashboard
2. Describe your agent in the text area
3. Review the generated blueprint
4. Customize any settings
5. Click **Deploy**

### Via CLI

```bash
chronos spark "A research assistant that searches academic papers,
summarizes findings, and maintains a knowledge base"
```

Output:

```yaml
# Generated by Spark
name: academic-researcher
description: Research assistant for academic papers
model: gemini-2.0-flash
temperature: 0.3

system_prompt: |
  You are an academic research assistant. You help users find,
  analyze, and synthesize academic papers. You maintain a knowledge
  base of previously researched topics and build on prior findings.
  Always cite papers with title, authors, year, and DOI when available.

memory:
  type: persistent
  vector_search: true

tools:
  - name: search_papers
    builtin: true
    config:
      sources: [arxiv, semantic_scholar, google_scholar]
  - name: summarize_paper
    builtin: true
  - name: save_to_knowledge_base
    builtin: true
  - name: query_knowledge_base
    builtin: true
```

### Via API

```python
from chronos import ChronosClient

client = ChronosClient(api_key="your_key")

blueprint = client.spark.generate(
    description="A customer support agent that handles refunds and billing questions",
    options={
        "model": "gemini-2.0-flash",
        "channels": ["api", "telegram"],
        "memory": True
    }
)

# Review the blueprint
print(blueprint.yaml())

# Deploy directly
agent = client.spark.deploy(blueprint)
print(f"Agent live at: {agent.endpoint}")
```

## Iterative Refinement

Spark supports multi-turn conversations to refine your agent:

```
You: "Make a support agent for an e-commerce store"
Spark: [generates blueprint]

You: "Add a tool that checks order shipping status via our API"
Spark: [adds shipping_status tool with API integration]

You: "The agent should be bilingual — English and French"
Spark: [updates system prompt for bilingual support]

You: "Deploy it"
Spark: ✓ Deployed to https://api.mohex.org/agents/ecom-support
```

## Blueprint Components

When Spark generates a blueprint, it configures:

| Component | What Spark Decides |
|-----------|--------------------|
| **Model** | Best LLM for the use case |
| **System Prompt** | Personality, instructions, guardrails |
| **Tools** | Required built-in + custom tool definitions |
| **Memory** | Type and configuration based on needs |
| **Channels** | Recommended communication channels |
| **Guardrails** | Safety filters and response boundaries |

## Tips for Better Results

1. **Be specific** — "A support agent for a SaaS billing tool" beats "A helpful agent"
2. **Mention integrations** — "Connected to our Stripe account" helps Spark configure tools
3. **Define the audience** — "For enterprise customers" changes tone and complexity
4. **Include constraints** — "Must never discuss competitor pricing" adds guardrails
